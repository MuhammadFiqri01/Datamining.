04 — Parameter

Penentuan Threshold

Penentuan Support & Confidence

Minimum Support

0.05  = 5%

min_support=0.05
Support(A) = freq(A) / N_transaksi

Produk/kombinasi harus muncul di minimal 5% dari 500 transaksi ≈ ≥ 25 transaksi. Menyaring itemset yang terlalu jarang.

Minimum Lift Threshold

1.00  = Lift ≥ 1

min_lift_threshold=1.0
Lift(A→B) = Confidence(A→B) / Support(B)

Aturan asosiasi hanya ditampilkan jika Lift ≥ 1.0. Lift di bawah 1 berarti korelasi negatif — artinya A dan B justru cenderung tidak dibeli bersama.

Alasan Pemilihan Threshold

✦

Dataset kecil (133 transaksi) — threshold rendah agar pola tidak tersaring habis

✦

Support 5% cukup selektif namun masih menemukan pola bermakna pada dataset ini

✦

Lift ≥ 1.0 memastikan hanya aturan dengan korelasi positif yang ditampilkan

5- Algoritma & Alur Kerja

Alur Kerja Algoritma Apriori

Langkah-Langkah Iteratif

1
Scan Database

Hitung frekuensi setiap item tunggal. Buang item di bawah minSupport → L₁

2
Generate Kandidat (Apriori-Gen)

Gabungkan Lₖ₋₁ dengan dirinya sendiri → hasilkan Cₖ (kandidat k-itemset)

3
Prune (Anti-Monotone Property)

Hapus kandidat yang subset-nya tidak frequent. Ulangi langkah 1-3

4
Hasilkan Association Rules

Untuk tiap frequent itemset buat aturan A→B, hitung confidence & lift

Implementasi Python

# Apriori MANUAL — tanpa library mlxtend import itertools min_support = 0.05 # 5% min_count = min_support * num_transactions # Hitung C1 → F1 (1-itemset) item_counts = {} for tx in transactions: for item in tx: item_counts[item] = item_counts.get(item,0)+1 # Filter & iterasi K-itemset via union frozenset df_rules = df_rules.sort_values("Lift Ratio", ascending=False)

Ilustrasi Logika — C++ Pseudocode

vector<Itemset> apriori(DB db, float minSup) { auto L1 = scanAndFilter(db, minSup); vector<Itemset> all = L1; for (int k=2; !all.empty(); k++) { // generate candidate k-itemsets auto Ck = aprioriGen(all, k); // prune: subset tidak frequent dihapus auto Lk = countAndPrune(db, Ck, minSup); if (Lk.empty()) break; all.insert(all.end(), Lk.begin(), Lk.end()); } return all; } float calcLift(Rule r, DB db) { return r.confidence / support(r.consequent, db); } bool isFrequent(Itemset s, DB db, float minSup) { return support(s, db) >= minSup; // Anti-monotone }
06 — Metrik Evaluasi

Evaluasi Aturan

Memahami Support · Confidence · Lift

Support

Support(A) = |txn mengandung A| / |Total txn|

Mengukur seberapa sering suatu itemset muncul dalam dataset. Nilai tinggi berarti itemset populer.

Confidence

Confidence(A→B) = Support(A∪B) / Support(A)

Keyakinan bahwa jika seseorang membeli A, ia juga akan membeli B. Rentang 0–1.

Lift Ratio

Lift(A→B) = Confidence(A→B) / Support(B)
Lift < 1Korelasi negatif
Lift = 1Independen
Lift > 1Korelasi positif — semakin tinggi semakin kuat
